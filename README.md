ゲーム『Harvest and Cook』技術仕様書：動作環境・実行方法・ファイル構成

1. プロジェクト概要

本プロジェクトは、PythonおよびPygameフレームワークを用いて開発された、農業と料理を基軸とするシミュレーションゲームです。 システム設計の核心は「経済サイクル（種子の購入 → 栽培・収穫 → 調理 → 出荷・換金）」のループにあり、プレイヤーはリソース管理とタイミング精度の双方が求められるメカニズムを体験します。
主要ゲームサイクル:
農業システム: タイルベースのグリッド上で行われる3種類の作物（カブ、トマト、ナス）の栽培。
調理システム: 収穫物を加工し、価値を高めるタイミング制御型ミニゲーム。
出荷・経済システム: グローバル座標に配置された出荷オブジェクトによる経済活動の完結。
ステート管理: game_state 変数を用いた、農業（farm）と料理（cooking）のシーン遷移制御。

8. 実装上の技術的特徴
本プログラムにおける重要な実装ポイントを解説します。
インデックスベースの選択ロジック: current_type_index 変数を用いることで、VEGETABLE_TYPES リスト内を循環的に参照しています。これにより、将来的な作物の追加に対してコードの変更を最小限に抑える拡張性を確保しています。
座標系と中心配置: プレイヤーの初期座標 [400, 300] は、解像度 800x600 の完全な中央値に設定されています。これは、プレイヤーが農地全域へ効率的にアクセスできるように意図された設計です。
衝突判定（Collision Detection）: shipping_box_rect という静的な pygame.Rect オブジェクトを定義し、プレイヤーの get_rect() との交差判定を行うことで、特定の場所でのみ「出荷」のアクションをトリガーする空間的なインタラクションを実現しています。
ミニゲームのメカニズム: 調理システムは、random.randint によって動的に生成された target 座標に対し、定速移動する bar_pos が重なった瞬間の入力を検知する、タイミング依存型の衝突チェックロジックとして実装されています。
フレームレートと描画の最適化: pygame.time.Clock によるFPS制御と、pygame.display.flip()（または update()）による画面更新により、ハードウェアに依存しない一定のゲームスピードと安定したレンダリングを保証しています。
